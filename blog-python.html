# Python을 주력 언어로 선택한 이유

백엔드 개발 및 데이터베이스 아키텍처에 관심이 많은 저에게 Python은 데이터를 다루고 외부 API와 연동하는 데 있어 가장 직관적이고 강력한 도구였습니다. 풍부한 라이브러리 생태계를 바탕으로 단순한 스크립트 작성을 넘어, 시스템의 자동화와 백엔드 로직 설계의 기본기를 탄탄히 다질 수 있었습니다.

# 핵심 학습 내용 및 코드 리뷰

API 연동 시 발생할 수 있는 네트워크 불안정성에 대비하여 방어적 프로그래밍(Defensive Programming)을 습관화하고 있습니다. 다음은 환율 API 연동 프로젝트 진행 시 직접 구현했던 `try-except` 기반의 안전한 API 호출 구조입니다.

import requests
import json

def fetch_exchange_rate(api_url):
    try:
        # 타임아웃 5초 설정으로 서버 무한 대기 방지
        response = requests.get(api_url, timeout=5.0)
        response.raise_for_status() # 200 OK가 아닐 경우 예외 발생
        
        data = response.json()
        return data

    except requests.exceptions.Timeout:
        print("API 서버 응답 시간이 초과되었습니다.")
    except requests.exceptions.HTTPError as err:
        print(f"HTTP 에러 발생: {err}")
    except json.JSONDecodeError:
        print("JSON 파싱에 실패했습니다. 데이터 형식을 확인하세요.")
    
    return None

# 성장을 위한 트러블슈팅 💡

Q. 대량의 데이터 처리 시 반복문 성능 저하 문제

상황: 오픈소스 리팩토링 과정에서 수천 줄의 텍스트 데이터를 List에 단순 `append()`로 추가하다 보니 메모리 사용량이 늘어나고 속도가 느려지는 현상 발견.

해결 방안: 파이썬의 제너레이터(Generator)와 `yield` 키워드를 학습하여 적용했습니다. 데이터를 한 번에 메모리에 올리지 않고 필요할 때마다 하나씩 생성하여 메모리 낭비를 최적화했습니다. 이를 통해 파이썬의 메모리 관리 기법에 대해 더 깊이 이해할 수 있었습니다.
