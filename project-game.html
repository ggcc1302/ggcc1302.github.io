무협 테마 1인 인디 게임 개발
(Steam 출시 목표)

1. 프로젝트 개요

전 세계적인 인기를 끈 로그라이크 액션 게임 뱀파이어 서바이버(Vampire Survivors) 스타일의 핵심 재미 요소에 독창적인 '무협' 테마를 결합한 PC 게임 프로젝트입니다. 새로운 개발 환경인 Unity 엔진과 C# 언어를 주도적으로 학습하며, 기획, 아트 에셋 매핑, 게임 로직 구현, 빌드까지 1인 인디 게임 개발의 전체 파이프라인을 경험하고 최종적으로 Steam 플랫폼 출시를 목표로 진행 중입니다.

2. 핵심 개발 내용 및 학습 (Unity / C#)

  • Unity 엔진 및 C# 습득: 컴포넌트 기반 설계(ECS 기초), 오브젝트 풀링, 충돌 처리, 코루틴을 활용한 비동기 로직 등 게임 개발 핵심 개념을 스터디하며 실제 코드로 구현
  • 게임 핵심 메커니즘 구축: 대량의 몬스터 웨이브(Enemy Spawning) 시스템, 플레이어 스킬 트리에 따른 무공 승급 로직, 로그라이크 방식의 무작위 강화 선택 시스템 구현
  • (향후 계획) DB/백엔드 연동: 게임 내 플레이 기록(스코어, 클리어 타임)을 저장하고 관리하기 위한 기본적인 데이터베이스 저장 시스템(또는 간단한 REST API 서버)을 연동하여 백엔드 개발 역량과 연결 예정

3. 트러블슈팅 및 도전 과제 (Challenges) 💡

이슈: 대량의 몬스터 동시 출현 시 발생하는 프레임 드랍(성능 저하)

발생 상황: '서바이버' 장르 특성상 수백 마리의 몬스터가 한 화면에 등장할 때, 매번 몬스터 오브젝트를 Instantiate()(생성)하고 Destroy()(파괴)하는 방식이 CPU에 큰 과부하를 주어 게임이 끊기는 현상 발생.

해결 과정 및 배운 점: 단순한 기능 구현을 넘어 메모리 관리 및 최적화의 중요성을 깊이 깨달았습니다.

이를 해결하기 위해 오브젝트 풀링(Object Pooling) 기법을 스터디하여 미리 필요한 만큼의 몬스터 오브젝트를 메모리에 할당해두고, 필요할 때마다 활성화/비활성화시켜 재사용하는 구조로 변경했습니다. 결과적으로 CPU 사용량을 대폭 줄여 안정적인 60 FPS(프레임)를 확보할 수 있었습니다.
