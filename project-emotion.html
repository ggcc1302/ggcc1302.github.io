Capstone Design Project

AI 기반 감정 분석 및
공감 상호작용 서비스 '감정창고'

1. 프로젝트 개요

단순한 텍스트 기록을 넘어 AI가 사용자의 감정을 분석하고 따뜻한 위로와 피드백을 건네는 지능형 일기 서비스입니다. 기존 일기 앱들의 단방향적인 텍스트 저장 기능을 개선하여, LLM(Gemini)을 활용해 맥락을 이해하고 과거 기록을 바탕으로 개인화된 멘탈 케어를 제공하는 것을 목표로 합니다.

2. 담당 역할 및 핵심 기여 (Backend / DB)

팀 내에서 데이터 기획, API 엔지니어링, 데이터베이스 구축을 총괄하고 있습니다.

  • 데이터베이스 구조 설계: 사용자 정보, 일기 원문, 분석된 복합 감정 수치를 효율적으로 저장하고 조회하기 위한 MySQL 관계형 DB 스키마 설계 (ERD 구축)
  • Gemini API 연동: 비정형 텍스트 데이터에서 감정 키워드와 수치를 정교하게 추출해 내기 위한 프롬프트 엔지니어링 및 파이프라인 구축
  • 맞춤형 큐레이션 로직 설계: AI가 도출한 감정 분석 결과를 기반으로 사용자에게 적합한 콘텐츠(음악, 활동 등)를 매핑하는 알고리즘 기획

3. 트러블슈팅 (Troubleshooting) 💡

이슈: LLM API 호출에 따른 응답 지연 (Latency) 문제

원인 분석: 사용자가 일기를 저장할 때마다 Gemini API를 동기적으로 호출하여 감정 분석 결과를 기다리게 되면, 서버 응답 시간이 길어져 사용자 경험(UX)이 크게 저하되는 문제가 발생할 수 있습니다.

해결 과정 및 고민: 단순 저장은 즉각적으로 DB에 반영하되, 감정 분석 과정은 비동기(Asynchronous) 방식이나 백그라운드 태스크로 분리하여 처리하는 아키텍처를 검토 및 적용 중입니다. 이를 통해 일기 저장의 안정성을 확보하고 데이터 분석 대기 시간을 시각적으로 완화할 수 있도록 프론트엔드 팀과 협의하여 로직을 최적화하고 있습니다.
