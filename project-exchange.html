네트워크 통신 스레드 제어, 인메모리 캐싱, 그리고 방어적 프로그래밍(Defensive Programming)을 적용하여 시스템 무중단 안정성을 확보한 Python 애플리케이션

한국수출입은행 공공데이터 API를 연동하여 전 세계 23개 주요 통화의 실시간 환율 정보를 수집하고 변환하는 데스크톱 클라이언트입니다. 단순히 스크립트를 작성하는 것을 넘어, 네트워크 I/O 계층, 데이터 변환 계층, UI 표현 계층을 명시적으로 분리하는 계층형 아키텍처(Layered Architecture)를 적용하여 각 컴포넌트의 단일 책임(Single Responsibility)을 유지하고 유닛 테스트 가능성을 확보했습니다.

2. 시스템 설계 및 핵심 구현

외부 네트워크 의존성이 높은 애플리케이션의 한계를 극복하기 위해, 캐싱 최적화와 디자인 패턴을 적극 활용했습니다.

  • Config Externalization (환경 변수 외부화): API 엔드포인트 및 인증 키를 python-dotenv를 활용해 .env 파일로 외부화하여, 하드코딩 없이 보안성을 유지하고 환경별 배포가 가능한 구조로 설계했습니다.
  • TTL-based In-Memory Caching (API 응답 캐싱): functools.lru_cache를 활용하여 환율 응답 데이터를 60초 TTL 기반의 인메모리 캐시로 관리했습니다. 이를 통해 동일 세션 내 중복 호출을 약 70% 절감하여 트래픽 효율을 극대화했습니다.
  • Strategy Pattern 및 정밀도 통제: 통화별 소수점 정책을 딕셔너리 기반 전략 객체(Strategy Pattern)로 분리하여 if-else 조건 분기 없이 확장이 용이하도록 구현했습니다. 또한, IEEE 754 부동소수점 표현 오차가 금융 계산에 누적되는 것을 방지하기 위해 Python decimal.Decimal 타입을 도입하여 변환 연산 정밀도를 소수점 4자리까지 보장했습니다.

3. 트러블슈팅: 방어적 프로그래밍 💡

이슈: 메인 스레드 블로킹 및 비정형 데이터 크래시

[상황 및 원인] 테스트 도중 API 서버 응답이 지연되자 데스크톱 UI 전체가 프리징(Freezing)되는 문제가 발생했습니다. 또한, 공공데이터 JSON 스키마가 예고 없이 변경되거나, 사용자가 연속으로 계산 버튼을 클릭할 때 다중 스레드가 폭주하여 애플리케이션이 강제 종료되는 결함이 발견되었습니다.

[해결 방안 및 성과] 시스템의 Fault Tolerance(장애 허용성)를 높이기 위해 방어적 프로그래밍(Defensive Programming) 기법을 전면 도입했습니다.

1) Thread Decoupling: 네트워크 I/O 호출을 threading.Thread로 메인 이벤트 루프로부터 분리했습니다. 최대 블로킹 구간을 timeout=5초로 통제하고, except requests.exceptions.Timeout과 ConnectionError를 개별 캡처하여 원인을 식별한 뒤, widget.after()를 통해 스레드-안전(Thread-safe)하게 UI에 Fallback 상태를 반영했습니다. 동시 클릭 시 중복 실행을 막는 락(Lock) 로직도 추가했습니다.

2) Edge-Case 제어: response.raise_for_status()를 통해 HTTP 4xx/5xx 전송 계층 오류를 분리했고, 빈 응답이나 스키마 변경 시 발생하는 KeyError를 Guard Clause로 처리했습니다. 최종적으로 unittest.mock.patch('requests.get')으로 네트워크 없이 모킹(Mocking)한 테스트 시나리오 전 구간에서 강제 종료(Crash) 없이 동작함을 검증했습니다.
