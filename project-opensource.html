오픈소스 프로그램
기능 개선 및 커스터마이징

1. 프로젝트 개요

기존에 배포된 번역 오픈소스 프로그램을 직접 사용하며 느꼈던 불편함을 해소하기 위해, 소스코드를 클론하여 내부 로직을 분석하고 실무 작업 효율을 높이는 신규 기능들을 직접 추가한 프로젝트입니다.

2. 주요 개선 사항 및 기여

  • 데이터 내보내기/가져오기(I/O) 추가: 번역된 텍스트 데이터를 별도의 파일로 추출하고 다시 불러올 수 있는 로직을 구현하여 데이터 관리의 연속성 확보
  • 단축키 기반 배치(Batch) 프로세싱: 마우스 클릭 위주의 기존 UI에 키보드 단축키 시스템을 매핑하여 대량 데이터 편집 시의 작업 속도를 대폭 개선
  • 코드 리딩 및 구조 파악: 문서화가 부족한 타인의 파이썬 코드를 디버깅하며 데이터 흐름을 추적하고, 기존 로직과의 충돌을 피하는 안전한 리팩토링 수행

3. 트러블슈팅 (Troubleshooting) 💡

이슈: 레거시 코드와의 의존성 충돌 및 UI 이벤트 꼬임 현상

발생 상황: 새로운 단축키 이벤트와 텍스트 내보내기 로직을 추가하는 과정에서, 기존 GUI 프레임워크가 가지고 있던 이벤트 루프와 충돌하여 프로그램이 응답 없음에 빠지는 문제 발생.

해결 과정 및 배운 점: 단순히 기능을 덧붙이는 것이 아니라, 기존 코드가 어떤 구조(Class/Method)로 설계되었는지 먼저 완벽히 이해하는 것이 중요함을 깨달았습니다.

디버거(Debugger)를 활용해 이벤트 발생 시점과 메모리 상의 변수 변화를 한 줄씩 추적(Step-over)하였고, 기존 메인 루프를 방해하지 않도록 별도의 콜백 함수로 기능을 분리하여 안정적으로 통합하는 데 성공했습니다. 이 과정을 통해 유지보수하기 좋은 코드 설계의 중요성을 체감했습니다.
