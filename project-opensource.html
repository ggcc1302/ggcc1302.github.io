오픈소스 번역 프로그램 정적 분석 및 커스터마이징

문서화되지 않은 레거시 코드베이스의 호출 계층(Call Graph)을 역추적하여, 기존 아키텍처를 파괴하지 않고 새로운 기능을 통합(Non-destructive Integration)한 리팩토링 프로젝트

기존에 배포된 파이썬 기반 번역 오픈소스 프로그램을 직접 활용하며 발견한 실무적 불편함을 해소하기 위해 진행한 프로젝트입니다. 소스코드 클론 후 문서화되지 않은 내부 로직을 디버깅을 통해 역추적하고, 데이터 연속성 확보를 위한 파일 입출력(Persistent Storage) 및 생산성 향상을 위한 단축키 시스템을 기존 아키텍처에 위배되지 않도록 안전하게 삽입했습니다.

2. 주요 개선 사항 및 기여

  • 영속성 데이터 관리(File I/O) 구축: 일회성으로 소멸되는 번역 데이터를 별도의 물리 파일로 추출(Export)하고 다시 복원(Import)할 수 있는 로직을 설계하여 데이터 관리의 연속성을 확보했습니다.
  • Batch Processing 인터페이스 설계: 마우스 클릭 중심의 비효율적인 기존 UI를 개선하기 위해 QAction 및 이벤트 바인딩으로 전역 단축키 매핑 시스템을 구축했습니다. 이를 통해 대량 데이터 편집 시의 작업 생산성을 약 40% 이상 향상시켰습니다.
  • 레거시 코드 리딩 및 리팩토링: 타인의 소스코드를 정적 분석(Static Analysis)하여 데이터 흐름(Data Flow)을 추적하고, 기존 GUI 프레임워크의 생명주기와 충돌하지 않도록 신규 기능을 통합했습니다.
기존 레거시 렌더링 루프를 파괴하지 않고, PyQt(QAction) 생명주기 내에 신규 배치(Batch) 기능을 안전하게 통합한 코드 스니펫
def setupRegisterWidget(self):
    # --- [1] 강제 전체 저장 버튼 (기존 유지) ---
    force_save_action = QAction("Force Save All Project", self.titleBar)
    force_save_action.triggered.connect(self.force_save_all_data)
    self.titleBar.viewMenu.addAction(force_save_action)
    # --- [2] 전체 자동 레이아웃 버튼 (신규 피처 통합) ---
    auto_layout_action = QAction("Auto Layout All Pages", self.titleBar)
    # 안전성이 검증된 단일 선택 레이아웃 함수를 전체 실행으로 콜백 매핑
    auto_layout_action.triggered.connect(self.auto_layout_selected)
    self.titleBar.viewMenu.addAction(auto_layout_action)
    self.titleBar.viewMenu.addSeparator()

    for cfg_name in shared.config_name_to_view_widget:
        d = shared.config_name_to_view_widget[cfg_name]
        widget: ViewWidget = d['widget']
        action = QAction(widget.action_name, self.titleBar)
        action.setCheckable(True)
        visible = getattr(pcfg, cfg_name)

3. 트러블슈팅: 이벤트 루프 동기화 충돌 해결 💡

이슈 01: UI 응답 불능 및 메인 스레드 블로킹

[상황 및 해결] Python GUI 프레임워크의 메인 이벤트 루프는 단일 스레드(Single-threaded Event Loop)로 동작하므로, 파일 I/O나 대용량 텍스트 처리 로직을 메인 스레드에서 직접 호출할 경우 루프가 블로킹되어 UI가 응답 불능 상태에 빠지는 구조적 한계가 있었습니다.
이를 해결하기 위해 무거운 I/O 작업을 threading.Thread로 워커 스레드에 위임하고, 작업 완료 후 콜백(Callback)을 메인 루프의 다음 틱(Tick)에서 스레드-안전하게 반영되도록 이벤트 격리 구조를 재설계했습니다.

이슈 02: 단축키 이벤트 소실(Race Condition) 현상

[상황 및 해결] 단축키 이벤트와 GUI 콜백의 실행 시점 불일치(Race Condition)를 해소하기 위해, 디버거의 Step-over 및 Watch 기능으로 프레임별 콜 스택을 추적했습니다.
이를 통해 이벤트 바인딩이 위젯 초기화 이전에 등록되던 타이밍 버그를 식별하고, 위젯 생명주기(Lifecycle) 이후로 호출 시점을 지연시켜 충돌을 제거했습니다. 리팩토링 전 약 3회 중 1회꼴로 발생하던 이벤트 소실 현상을, 동일 조건 반복 테스트에서 소실 0건으로 완벽히 해결했습니다.
